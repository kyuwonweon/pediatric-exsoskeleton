# State Machine Reference

The state machine engine loads and executes `.psm` binary files generated by CAPS. It runs on a timer ISR at 100 Hz and is only active on PRIMARY_CONTROLLER devices (Hybrid Knee, Fox Knee, Polycentric Ankle Only).

**Source:** `state_machine.py` (162 lines), `sm_assembler_functions.py` (1246 lines)

---

## Overview

- `.psm` files are created offline by a CAPS slave script (`create_psm.py`)
- The file is placed in the microcontroller's root directory as `uPythonStateMachine.psm`
- At boot, `HybridLeg.import_pyb_sm()` loads the file and creates a `StateMachine` instance
- The state machine is driven by sensor data in `sm_channels_current` (24-element float array)
- All runtime operations use ARM assembly functions for deterministic timing

---

## .psm Binary File Format

The file is read sequentially using `struct.unpack`:

| Offset | Type | Field | Description |
|--------|------|-------|-------------|
| 0 | `int16` | max_states | Number of states |
| 2 | `int16` | DOFs | Number of degrees of freedom |
| 4 | `int16` | starting_state | Initial state ID |
| 6 | `int16` | max_events | Total event count |
| 8 | `int16` | active_events | Currently active event count |

### Event Definitions (repeated `max_events` times)

| Type | Field | Description |
|------|-------|-------------|
| `int16` | state_from | Source state |
| `int16` | state_to | Destination state |
| `int16` | event_priority | Priority level |
| `int16` | event_enabled | 1 = enabled, 0 = disabled |

### State Definitions (repeated `max_states` times)

| Type | Field | Description |
|------|-------|-------------|
| `int16` | active_state | State ID |
| `int16` | num_list_events | Number of event list entries |
| `float[num_list_events]` | event_list_array | Event transition conditions |
| `int16` | num_state_events | Number of state output entries |
| `float[num_state_events]` | state_list_array | State output/DOF calculations |

### DOF ID Map (at end of file)

| Type | Field | Description |
|------|-------|-------------|
| `float[DOFs]` | dof_id_map | Maps DOF indices to IDs |

## StateMachine Class

### Constructor (`__init__`)

1. Opens and parses `.psm` file
2. Creates pre-allocated arrays for all state machine data:
   - `event_id`, `state_from`, `state_to`, `event_priority`, `event_enabled`
   - `event_list_arrays` (one per state)
   - `state_list_arrays` (one per state)
   - `dof_id_map`, `dof_outputs`
3. Builds `state_dictionary` mapping state IDs to array indices
4. Creates sensor address arrays for assembly function access:
   - `current_chans` — current sensor values (24 floats)
   - `prev_chans` — previous sensor values
   - `slope_chans` — rate of change
   - `max_chans`, `min_chans` — observed extremes
   - `state_entry_chans` — values at state entry
5. Creates `timing_vars` array: `[time_in_state, frame_counter, manual_override, state_change_var]`

### Key Attributes

| Attribute | Type | Description |
|-----------|------|-------------|
| `active_state` | `array('i', [state_id])` | Current state (accessible from assembly) |
| `current_state` | `int` | Current state ID |
| `sm_channels_current` | `array('f', 24)` | Input sensor data |
| `dof_outputs` | `array('f', DOFs)` | DOF output values |
| `timing_vars` | `array('f', 4)` | [time_in_state, frame_counter, manual_override, state_change] |
| `sm_time_val` | `int` | Execution time of last tick (microseconds) |

### Methods

| Method | Description |
|--------|-------------|
| `start_machine()` | Enable state machine execution |
| `stop_machine()` | Disable state machine execution |
| `attach_timer(timer)` | Register timer callback for `execute_state_machine` |
| `execute_state_machine(timer)` | Main ISR callback (100 Hz) |

## execute_state_machine ISR (100 Hz)

**Source:** `state_machine.py:134`

### Execution Flow

```
1. Calculate time_in_state = ticks_ms() - state_start_time
2. Increment state_frame_counter
3. Save old_state = active_state[0]
4. sm_assembler_functions.update_sm_input_vars()
   → Updates timing, slopes, min/max from sensor data
5. sm_assembler_functions.check_sub_events()
   → Evaluates event transition conditions for current state
6. sm_assembler_functions.check_states()
   → Processes state outputs and DOF calculations
7. Update current_state from active_state[0]
8. IF state changed:
   a. Reset state_start_time
   b. Reset state_frame_counter
   c. sm_assembler_functions.state_change_cleanup()
      → Resets prev_chans, slope_chans, extremes
   d. Update sm_dictionary_index for new state
   e. Update subevent_inputs pointer to new state's event_list_array
```

### Assembly Functions (`sm_assembler_functions.py`)

All operations are implemented in ARM assembly (via `@micropython.asm_thumb` or `@micropython.viper`) for deterministic timing:

| Function | Description |
|----------|-------------|
| `update_sm_input_vars(variable_indexes, num_chans, time_in_state, frame_counter)` | Update sensor slopes, min/max, timing |
| `check_sub_events(subevent_inputs, end_address, event_output, sensor_addresses)` | Evaluate event transition conditions |
| `check_states(state_list_array, sensor_addresses, offset, sm_state_dofs)` | Process state outputs and DOF values |
| `state_change_cleanup(sensor_addresses, num_chans)` | Reset tracking variables on state transition |

## Integration with HybridLeg

### Initialization (`hybrid_leg.py:138-170`)

```python
# Only for PRIMARY_CONTROLLER devices
if self.device_parameters.PRIMARY_CONTROLLER:
    self.sm_channels_current = array.array('f', [0] * 24)
    # ... create scaling arrays, max/min/slope arrays ...

    file_name = "uPythonStateMachine.psm"
    if file_name in os.listdir():
        self.sm = StateMachine(
            self.sm_channels_current, sm_channels_max,
            sm_channels_min, slope_chans, file_name
        )
        self.sm_loaded = True

    # Timer attachment happens in HardwareManager
    self.sm.attach_timer(self.timer_state_machine)
    self.sm.start_machine()
```

### Sensor Channel Population

`sm_channels_current` is populated by multiple ISRs:

| Channel Range | Source | ISR |
|---------------|--------|-----|
| Joint angle/speed | Servo encoder | `isr_joint_controller` |
| Load cell data | CAN message | CAN FIFO1 callback |
| Key fob value | CAN message (CAPS1) | CAN FIFO0 callback |
| IMU data | BNO085 SPI | `tick_imu` |

### State Output to CAPS

In `tick_caps` (odd ticks, knee devices):
```python
if self.sm_loaded:
    PARAM_DICT['ACTIVE_STATE'][0] = float(self.sm.active_state[0])
# Sent in can_out_mes2 (CAPS14_DATA, NID 0x021)
```
